# 🏗️ Table Generation Prompt Library

This document captures all prompts used to design and generate base tables for the insurance claims data platform.

---

## 🧠 Foundation Prompts

### 1.
I want you to design a set of base tables for an insurance claims system. The tables should support analytics use cases, including claims, policies, customers, payments, and events. Ensure the structure is scalable and normalized.

### 2.
Create SQL DDL statements in Snowflake for all core tables. Use appropriate data types and ensure naming conventions are clean and enterprise-ready.

### 3.
Make sure every table has:
- Primary key
- Audit columns (created_date, updated_date)
- Proper naming conventions (snake_case or uppercase)

### 4.
Design tables in such a way that they can later support a star schema model.

---

## 📊 Claims Table

### 5.
Create a FACT_CLAIMS table that captures all claim-level information such as:
- claim_id
- policy_id
- claim_amount
- claim_status
- dates

### 6.
Ensure this table supports both transactional and analytical use cases.

### 7.
Add fields for fraud indicators, litigation flags, and weather-related indicators.

---

## 📦 Dimension Tables

### 8.
Create dimension tables for:
- Claim Type
- Geography
- Date
- Policy
- Loss Cause

### 9.
Ensure each dimension table has a surrogate key.

### 10.
Make sure dimension tables are denormalized and optimized for analytics.

---

## 📅 Date Dimension

### 11.
Generate a DIM_DATE table with:
- date_key
- full date
- year, month, quarter
- fiscal attributes

### 12.
Ensure it supports time-series analytics.

---

## 🌍 Geography Dimension

### 13.
Create a DIM_GEOGRAPHY table with:
- region
- country
- state
- city

### 14.
Ensure hierarchical relationships are clear.

---

## ⚙️ Best Practices

### 15.
Ensure all tables are Snowflake-optimized (clustering if needed).

### 16.
Avoid over-normalization.

### 17.
Ensure naming consistency across all tables.

---

## 🏁 Outcome

These prompts generate a **strong foundation layer** for analytics-ready data.
