# ⭐ Star Schema Modeling Prompt Library

This document captures prompts used to design a star schema model for analytics.

---

## 🧠 Modeling Approach

### 1.
Design a star schema for insurance claims analytics.

### 2.
Identify fact tables and dimension tables clearly.

### 3.
Ensure schema supports BI tools and dashboarding.

---

## 📊 Fact Table Design

### 4.
Create a FACT_CLAIMS table as the central fact table.

### 5.
Include measures like:
- claim amount
- paid amount
- incurred loss

### 6.
Ensure foreign keys link to dimension tables.

---

## 📦 Dimension Design

### 7.
Create dimension tables for:
- Date
- Geography
- Claim Type
- Policy

### 8.
Ensure dimensions are descriptive and denormalized.

---

## 🔑 Keys

### 9.
Use surrogate keys for dimensions.

### 10.
Ensure all joins are based on keys, not text fields.

---

## 📐 Schema Design

### 11.
Ensure star schema structure:
- one central fact
- multiple surrounding dimensions

### 12.
Avoid snowflaking unless necessary.

---

## 🚀 Performance

### 13.
Optimize schema for query performance.

### 14.
Use clustering where required.

---

## 🏁 Outcome

These prompts create a **clean, scalable star schema** for analytics.
