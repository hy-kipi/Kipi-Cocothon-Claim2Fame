# 🧠 Semantic Layer Prompt Library

This document captures prompts used to build a semantic layer using Snowflake Cortex.

---

## 🧠 Objective

### 1.
Create a semantic layer that allows business users to query data using natural language.

### 2.
Abstract complexity of SQL from end users.

---

## 📊 Semantic View

### 3.
Create a semantic view that maps business-friendly terms to underlying tables.

### 4.
Ensure naming is intuitive (e.g., “Total Claims” instead of FACT_CLAIMS).

---

## 🔗 Mapping

### 5.
Map all fact and dimension tables into a unified semantic model.

### 6.
Ensure relationships are clearly defined.

---

## 🤖 Cortex Integration

### 7.
Enable Snowflake Cortex Analyst for querying the semantic view.

### 8.
Ensure responses are business-friendly, not SQL-heavy.

---

## 🧾 Context Definition

### 9.
Define schema context for AI:
- tables
- joins
- relationships

### 10.
Ensure AI understands how to generate queries correctly.

---

## 💬 Natural Language Queries

### 11.
Allow users to ask:
- “What are total claims this month?”
- “Which region has highest losses?”

### 12.
Ensure responses include insights, not just data.

---

## ⚙️ Optimization

### 13.
Ensure semantic layer is performant.

### 14.
Avoid unnecessary joins.

---

## 🏁 Outcome

These prompts enable a **business-friendly AI-powered data access layer**.
