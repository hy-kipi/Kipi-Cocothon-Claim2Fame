# 🔄 Data Pipeline Prompt Library

This document contains prompts used to design and implement data pipelines using Snowflake.

---

## 🧠 Initial Load

### 1.
Design a pipeline to load initial data into Snowflake tables from source systems.

### 2.
Ensure data is validated before loading into target tables.

### 3.
Create staging tables to hold raw data before transformation.

---

## ⚙️ Transformation Layer

### 4.
Transform raw data into structured format suitable for analytics.

### 5.
Ensure transformations include:
- data cleansing
- type casting
- null handling

### 6.
Create reusable SQL transformations.

---

## 🔁 Incremental Processing

### 7.
Now design an incremental data pipeline using Snowflake Streams.

### 8.
Use Streams to capture change data (CDC).

### 9.
Create Tasks to process incremental data at regular intervals.

### 10.
Ensure idempotency — pipeline should not duplicate data.

---

## ⏱️ Scheduling

### 11.
Set up Snowflake Tasks to run at scheduled intervals.

### 12.
Ensure dependencies between tasks are handled properly.

---

## 📊 Monitoring

### 13.
Add logging and monitoring for pipeline execution.

### 14.
Capture:
- success/failure
- row counts
- execution time

---

## 🚨 Error Handling

### 15.
Design retry mechanisms for failed pipeline runs.

### 16.
Capture failed records separately for analysis.

---

## 🏁 Outcome

These prompts help build a **robust, scalable, incremental data pipeline** using Snowflake.
